# AI 命理人格系统 MVP 内测验证清单

## 目标

本轮内测只验证 MVP 是否具备继续小规模用户测试的条件，不验证生产级收费上线能力。

## 参与角色

- 产品：确认流程、文案、付费前后权益边界。
- 测试/研发：执行环境验证、功能用例、异常用例。
- 命理顾问：复核排盘与标签解释是否可接受。
- 运营：收集用户反馈、截图、转化数据。

## 环境口径

- 使用开发/内测环境，不接真实线上流量。
- 允许启用 mock AI 和 mock 支付，但测试记录必须注明是否使用 mock。
- 不采集真实身份证、手机号等高敏信息。
- 出生信息可使用内测用户自愿提供数据或构造样本。

## 启动前门禁

在 `fate-gate` 目录执行：

```bash
npm run verify:internal
```

最低通过标准：

- TypeScript 类型检查通过。
- Vitest 单元测试通过。
- Next.js production build 通过。
- 如 build 仅因本机超时中断，需记录最后输出阶段和是否已完成页面生成。

## 核心路径用例

| 编号 | 用例 | 通过标准 |
| --- | --- | --- |
| IT-01 | 首次进入首页 | 页面可打开，表单可输入出生日期、时辰、地点 |
| IT-02 | 已知时辰生成摘要 | 30 秒内返回四柱、五行、人格标签、核心特质 |
| IT-03 | 未知时辰问卷 | 可打开问卷，选择后继续生成摘要 |
| IT-04 | 免费摘要分享 | 分享入口可用，无法原生分享时有兜底卡片/复制链路 |
| IT-05 | 解锁完整报告 | mock 支付成功后可生成并查看完整报告 |
| IT-06 | 我的报告 | 用户能看到已生成报告，空状态不报错 |
| IT-07 | 合盘入口 | 页面可访问，基础输入/结果流程无阻塞 |
| IT-08 | 健康检查 | `/api/v1/health/live` 和 `/api/v1/health/ready` 返回可解释状态 |

## 命理准确性抽样

本轮不要求达到 PRD 的 100 个样本标准，但需要建立样本表。

准生产口径已确定启用真太阳时。内测阶段支持任意地点名称解析，系统会先走地名解析再补充经纬度；如果仍需人工补录，也必须记录真太阳时计算元数据。

| 样本类型 | 数量 | 关注点 |
| --- | --- | --- |
| 普通日期 | 10 | 真太阳时修正后年月日时四柱基础正确 |
| 节气边界 | 5 | 真太阳时修正后月柱切换是否符合预期 |
| 子时边界 | 5 | 真太阳时修正后日柱/时柱边界处理 |
| 不同出生地 | 5 | 北京、上海、乌鲁木齐、拉萨、哈尔滨等经度差异 |
| 未知时辰 | 5 | 问卷结果与真太阳时策略不冲突 |

每个样本记录：输入时间、出生地点、经度、纬度、时区、真太阳时、修正分钟数、系统输出、人工参考输出、是否一致、差异说明。

严重错误定义：

- 未使用出生地经度却标记为已启用真太阳时。
- 修正后跨时辰但系统仍按原时辰排盘。
- 修正后跨日期但系统未处理日柱边界。
- 缓存命中错误导致不同地点得到同一排盘结果。

## 数据记录

每位内测用户记录：

- 来源渠道。
- 是否完成摘要生成。
- 是否点击解锁。
- 是否完成 mock 支付。
- 是否分享。
- 主观评分 1-5。
- 最困惑的一句话。
- 最愿意分享的一句话。

## 暂不作为本轮通过标准

- 真实微信支付成功率。
- 真实订阅续费。
- 多实例 Redis 缓存一致性。
- 生产级 JWT/微信登录。
- 大规模并发。
- 完整合规文档上线。

## Go / No-Go 判断

建议进入下一轮小规模真实用户测试的条件：

- 核心路径 IT-01 到 IT-05 全部通过。
- 单元测试和类型检查通过。
- 命理抽样严重错误为 0；非严重差异可解释。
- 至少 20 位内测用户完成摘要生成。
- 摘要完成率高于 70%。
- 主观评分平均不低于 3.8/5。
